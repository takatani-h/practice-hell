# PracticeHell

> 身につくまで、終わらない。

PracticeHell は、生成AIによる問題生成と、教室全体のリアルタイムな進捗確認を組み合わせた、習熟基準達成型の演習問題アプリです。

解答者は同じサーバーへ接続し、設定された基準を満たすまで問題を解き続けます。ファシリテータ（教師）は、問題の形式と習熟基準を設定し、解答者全員の進捗をリアルタイムで確認できます。

英語での概要: AI-generated adaptive exercises with real-time classroom progress tracking.

## コンセプト

単に数問を解いて終わるのではなく、知識や解法が身につくまで反復するための演習環境を目指します。

プロジェクト名の「Hell」には、「嫌になるほど繰り返す」「体に覚えさせる」という泥臭い演習のイメージを込めています。英語のフレーズでは `Practice the hell out of it.` に近い考え方です。

## 利用者

- **ファシリテータ（教師）**: 演習を準備し、習熟基準を設定して、参加者の進捗を確認する
- **解答者**: 演習へ参加し、習熟基準を満たすまで出題された問題を解く

## 基本フロー

1. ファシリテータが問題の形式を定義する
2. ファシリテータが総解答数と正答率の基準を設定する
3. アプリが指定された形式に従い、ランダムな問題と模範解答を生成AIで作成する
4. 解答者が共通のサーバーへ接続し、演習へ参加する
5. アプリが問題を提示し、解答を採点して履歴を保存する
6. 解答者は習熟基準を満たすまで問題を解き続ける
7. ファシリテータは解答者全員の進捗をリアルタイムで確認する

## 習熟基準

演習の完了には、次の両方を満たす必要があります。

- 総解答数が設定された閾値以上である
- 指数移動平均（EMA）による正答率が設定された閾値以上である

EMA は、各解答の正誤を正解 `1`、不正解 `0` として、次の式で更新する想定です。

```text
新しいEMA = α × 今回の正誤 + (1 - α) × 直前のEMA
```

これにより、単純な累積正答率だけでなく、直近の習熟状況をより強く反映できます。`α`、EMAの初期値、各閾値の具体的な設定方法は今後決定します。

## 主要機能

### ファシリテータ向け

- 演習セッションの作成
- 問題形式と生成条件の設定
- 総解答数とEMA正答率による習熟基準の設定
- 参加者ごとの解答数、正答状況、達成状況のリアルタイム確認
- 生成された問題、模範解答、採点結果の確認

### 解答者向け

- 演習セッションへの参加
- 問題の表示と解答の入力
- 採点結果の確認
- 現在の解答数、EMA正答率、達成状況の確認

### システム

- 定義された形式に基づく問題と模範解答の生成
- 解答の採点と履歴の保存
- 参加者の進捗のリアルタイム配信

## 未決事項

- 問題形式を定義する方法とデータ形式
- 対応する問題・解答形式と採点方法
- 生成した問題と模範解答の検証方法
- 認証方法と、ファシリテータ・解答者の権限管理
- EMAの初期値、係数 `α`、習熟基準の既定値
- 使用する技術スタックと運用環境

## 開発環境

```bash
# Python依存関係
uv sync

# フロントエンド依存関係
npm install

# API（別ターミナルで実行）
uv run alembic upgrade head
uv run uvicorn backend.main:app --reload

# React開発サーバー
npm run dev
```

既定では `LLM_PROVIDER=mock` のため、外部APIなしで問題生成を確認できます。設定値は
[.env.example](.env.example) を参照し、実際の秘密情報は `.env` ではなくsystemdの
`EnvironmentFile` など、VPS上のアクセス制限されたファイルへ保存してください。

## テスト

```bash
uv run --isolated pytest
npm test
npm run build
```

## VPSへの配置

1. `npm ci && npm run build` でReactを `dist/` へビルドする。
2. `uv sync --frozen --no-dev` でPython環境を作成する。
3. `uv run alembic upgrade head` でDBを更新する。
4. [deploy/practice-hell.service](deploy/practice-hell.service) を環境に合わせて配置する。
5. NginxまたはCaddyでHTTPSを終端し、`127.0.0.1:8000` へ転送する。

本番では `TEACHER_PASSWORD` と `SESSION_SECRET` を必ず変更し、`COOKIE_SECURE=true` を設定する。

OpenAI互換APIへ接続する場合は、次を設定する。

```dotenv
LLM_PROVIDER=openai-compatible
LLM_BASE_URL=https://api.openai.com
LLM_API_KEY=...
LLM_MODEL=利用するモデルID
```

互換APIアダプターは Chat Completions の構造化出力を利用する。接続先モデルがJSON Schema形式の
構造化出力に対応していることを確認する。
