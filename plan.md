# PracticeHell 実装計画

## 要点

React + TypeScript の画面と FastAPI + SQLite のAPIを単一VPSで動かし、解答者の演習フローを優先して開発する。

## MVP

- 教師は共通パスワードでログインし、数値問題の生成条件と習熟基準を設定する。
- AI生成問題は教師が問題文・正答・許容誤差を確認し、承認後に公開する。
- 解答者は参加コードと表示名で参加し、数値を回答する。
- 採点後に正誤、正答、解答数、EMA、達成状況を表示する。
- 習熟基準の既定値は20問以上、EMA 80%以上、`α = 0.2` とする。
- 問題は先行生成し、未出題問題が少なくなると承認待ちの補充問題を生成する。

## 技術構成

- フロントエンド: React、TypeScript、Vite
- バックエンド: FastAPI、SQLAlchemy、SQLite
- AI: OpenAI互換API。接続先、モデル、APIキーは環境変数で設定する。
- 配置: Reactの成果物をFastAPIから配信し、systemdとHTTPSリバースプロキシの内側で起動する。
